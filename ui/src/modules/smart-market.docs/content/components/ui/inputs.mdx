# Inputs

Form input components for collecting user data. Built on Radix primitives with consistent sizing, focus states, and accessibility.

```tsx
import { Input } from "@rio.js/ui-next/components/input"
import { Label } from "@rio.js/ui-next/components/label"
```

## Select

A dropdown for choosing from a predefined list of options.

```tsx
import {
  Select,
  SelectContent,
  SelectItem,
  SelectTrigger,
  SelectValue,
} from "@rio.js/ui-next/components/select"
```

<ComponentPreview>
  <div className="w-full max-w-sm space-y-2">
    <Label>Status</Label>
    <Select>
      <SelectTrigger>
        <SelectValue placeholder="Select status" />
      </SelectTrigger>
      <SelectContent>
        <SelectItem value="active">Active</SelectItem>
        <SelectItem value="inactive">Inactive</SelectItem>
        <SelectItem value="pending">Pending</SelectItem>
      </SelectContent>
    </Select>
  </div>
</ComponentPreview>

```tsx
<Select>
  <SelectTrigger>
    <SelectValue placeholder="Select status" />
  </SelectTrigger>
  <SelectContent>
    <SelectItem value="active">Active</SelectItem>
    <SelectItem value="inactive">Inactive</SelectItem>
    <SelectItem value="pending">Pending</SelectItem>
  </SelectContent>
</Select>
```

### When to use

- **Select** — When the user must choose exactly one option from a predefined list of 4+ items.
- **Radio Group** — When there are 2-5 options and all should be visible at once.
- **Combobox** — When the list is long and the user needs to search/filter.

## Switch

A toggle switch for on/off states. Prefer over checkboxes for settings that take immediate effect.

```tsx

```

<ComponentPreview>
  <div className="space-y-4">
    <div className="flex items-center gap-2">
      <Switch id="notifications" />
      <Label htmlFor="notifications">Enable notifications</Label>
    </div>
    <div className="flex items-center gap-2">
      <Switch id="dark-mode" defaultChecked />
      <Label htmlFor="dark-mode">Dark mode</Label>
    </div>
    <div className="flex items-center gap-2">
      <Switch id="disabled-switch" disabled />
      <Label htmlFor="disabled-switch">Disabled</Label>
    </div>
  </div>
</ComponentPreview>

```tsx
<Switch id="notifications" />
<Label htmlFor="notifications">Enable notifications</Label>
```

### When to use Switch vs Checkbox

- **Switch** — Instant effect (e.g., toggling a setting on/off). Binary on/off states.
- **Checkbox** — Deferred effect (e.g., form submission). Can represent "agree/disagree" or selection from a group.

## API Reference

### Input Props

Extends all native `<input>` HTML attributes.

| Prop          | Type                        | Default     | Description                                                             |
| ------------- | --------------------------- | ----------- | ----------------------------------------------------------------------- |
| `type`        | `string`                    | `"text"`    | HTML input type (`text`, `email`, `password`, `number`, `search`, etc.) |
| `size`        | `"sm" \| "default" \| "lg"` | `"default"` | Input height (`sm`: 32px, `default`: 36px, `lg`: 40px)                  |
| `pilled`      | `boolean`                   | `false`     | Renders with fully rounded corners (pill shape)                         |
| `icon`        | `boolean \| string`         | —           | Shows a left icon. Pass an icon name or `true`                          |
| `placeholder` | `string`                    | —           | Placeholder text                                                        |
| `disabled`    | `boolean`                   | `false`     | Disables the input                                                      |
| `className`   | `string`                    | —           | Additional CSS classes                                                  |

### Textarea Props

Extends all native `<textarea>` HTML attributes.

| Prop          | Type      | Default | Description                 |
| ------------- | --------- | ------- | --------------------------- |
| `rows`        | `number`  | —       | Number of visible text rows |
| `placeholder` | `string`  | —       | Placeholder text            |
| `disabled`    | `boolean` | `false` | Disables the textarea       |

### Select Props

| Prop            | Type                      | Default | Description                   |
| --------------- | ------------------------- | ------- | ----------------------------- |
| `value`         | `string`                  | —       | Controlled selected value     |
| `defaultValue`  | `string`                  | —       | Uncontrolled default value    |
| `onValueChange` | `(value: string) => void` | —       | Called when selection changes |
| `disabled`      | `boolean`                 | `false` | Disables the select           |

### SelectTrigger Props

| Prop        | Type                        | Default     | Description                                              |
| ----------- | --------------------------- | ----------- | -------------------------------------------------------- |
| `size`      | `"sm" \| "default" \| "lg"` | `"default"` | Trigger height (`sm`: 32px, `default`: 36px, `lg`: 40px) |
| `className` | `string`                    | —           | Additional CSS classes                                   |

### Checkbox Props

| Prop              | Type                         | Default | Description               |
| ----------------- | ---------------------------- | ------- | ------------------------- |
| `checked`         | `boolean`                    | —       | Controlled checked state  |
| `defaultChecked`  | `boolean`                    | `false` | Uncontrolled default      |
| `onCheckedChange` | `(checked: boolean) => void` | —       | Called when state changes |
| `disabled`        | `boolean`                    | `false` | Disables the checkbox     |

### Switch Props

| Prop              | Type                         | Default | Description               |
| ----------------- | ---------------------------- | ------- | ------------------------- |
| `checked`         | `boolean`                    | —       | Controlled on/off state   |
| `defaultChecked`  | `boolean`                    | `false` | Uncontrolled default      |
| `onCheckedChange` | `(checked: boolean) => void` | —       | Called when state changes |
| `disabled`        | `boolean`                    | `false` | Disables the switch       |

### RadioGroup Props

| Prop            | Type                         | Default      | Description                   |
| --------------- | ---------------------------- | ------------ | ----------------------------- |
| `value`         | `string`                     | —            | Controlled selected value     |
| `defaultValue`  | `string`                     | —            | Uncontrolled default          |
| `onValueChange` | `(value: string) => void`    | —            | Called when selection changes |
| `disabled`      | `boolean`                    | `false`      | Disables all radio items      |
| `orientation`   | `"horizontal" \| "vertical"` | `"vertical"` | Layout direction              |

### Slider Props

| Prop            | Type                        | Default | Description                   |
| --------------- | --------------------------- | ------- | ----------------------------- |
| `value`         | `number[]`                  | —       | Controlled value(s)           |
| `defaultValue`  | `number[]`                  | —       | Uncontrolled default value(s) |
| `onValueChange` | `(value: number[]) => void` | —       | Called when value changes     |
| `min`           | `number`                    | `0`     | Minimum value                 |
| `max`           | `number`                    | `100`   | Maximum value                 |
| `step`          | `number`                    | `1`     | Step increment                |
| `disabled`      | `boolean`                   | `false` | Disables the slider           |
