# Button

A versatile button component built on Radix primitives with support for multiple variants, sizes, icons, loading states, and composition via `asChild`.

```tsx
import { Button } from "@rio.js/ui/components/button"
```

---

## Variants

Buttons come in 9 style variants to communicate different levels of emphasis and intent.

<VariantGrid>
  <Button>Default</Button>
  <Button variant="primary">Primary</Button>
  <Button variant="secondary">Secondary</Button>
  <Button variant="destructive">Destructive</Button>
  <Button variant="danger">Danger</Button>
  <Button variant="success">Success</Button>
  <Button variant="outline">Outline</Button>
  <Button variant="ghost">Ghost</Button>
  <Button variant="link">Link</Button>
</VariantGrid>

```tsx
<Button>Default</Button>
<Button variant="primary">Primary</Button>
<Button variant="secondary">Secondary</Button>
<Button variant="destructive">Destructive</Button>
<Button variant="danger">Danger</Button>
<Button variant="success">Success</Button>
<Button variant="outline">Outline</Button>
<Button variant="ghost">Ghost</Button>
<Button variant="link">Link</Button>
```

### When to use each variant

- **Default / Primary** — Main call to action. Use for the most important action on the page.
- **Secondary** — Supporting actions alongside primary buttons.
- **Destructive / Danger** — Irreversible actions like delete, remove, or disconnect.
- **Success** — Positive confirmations like save, approve, or publish.
- **Outline** — Medium emphasis. Good for secondary actions in toolbars.
- **Ghost** — Low emphasis. Use for tertiary actions or within dense UI.
- **Link** — Navigational actions styled as inline links.

## Sizes

Four size options to fit different UI contexts.

<VariantGrid>
  <Button size="sm">Small</Button>
  <Button size="md">Medium</Button>
  <Button size="lg">Large</Button>
  <Button size="icon"><span className="icon-[ph--plus-bold]" /></Button>
</VariantGrid>

```tsx
<Button size="sm">Small</Button>
<Button size="md">Medium</Button>
<Button size="lg">Large</Button>
<Button size="icon">
  <span className="icon-[ph--plus-bold]" />
</Button>
```

| Size | Height | Use case |
|------|--------|----------|
| `sm` | 32px | Compact UI, table rows, toolbars |
| `md` | 36px | Default. Forms, dialogs, general actions |
| `lg` | 40px | Hero sections, prominent CTAs |
| `icon` | 36x36px | Icon-only buttons, toolbar icons |

## Loading State

Set `loading` to show a spinner and disable the button. The text is preserved for layout stability.

<VariantGrid>
  <Button loading>Saving...</Button>
  <Button variant="outline" loading>Loading</Button>
  <Button variant="destructive" loading>Deleting</Button>
</VariantGrid>

```tsx
<Button loading>Saving...</Button>
<Button variant="outline" loading>Loading</Button>
<Button variant="destructive" loading>Deleting</Button>
```

## With Tooltip

Use the `tooltip` prop to add a hover tooltip. Combine with `shortcut` to show a keyboard shortcut.

<VariantGrid>
  <Button tooltip="Create a new item" icon="icon-[ph--plus-bold]" size="icon" variant="outline" />
  <Button tooltip="Save changes" shortcut="⌘S">Save</Button>
</VariantGrid>

```tsx
<Button
  tooltip="Create a new item"
  icon="icon-[ph--plus-bold]"
  size="icon"
  variant="outline"
/>
<Button tooltip="Save changes" shortcut="⌘S">
  Save
</Button>
```

---

## Composition with asChild

Use `asChild` to render the button styles on a child element (e.g., a `Link`).

```tsx
import { Link } from "react-router"

<Button asChild variant="outline">
  <Link to="/settings">Go to Settings</Link>
</Button>
```

## Button Group

Group related buttons together. Adjacent borders merge automatically.

<VariantGrid>
  <ButtonGroup>
    <Button variant="outline">Left</Button>
    <Button variant="outline">Center</Button>
    <Button variant="outline">Right</Button>
  </ButtonGroup>
</VariantGrid>

```tsx
import { ButtonGroup } from "@rio.js/ui/components/button-group"

<ButtonGroup>
  <Button variant="outline">Left</Button>
  <Button variant="outline">Center</Button>
  <Button variant="outline">Right</Button>
</ButtonGroup>
```

## API Reference

### Button Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `Variant` | `"default"` | Visual style. One of `default`, `primary`, `secondary`, `destructive`, `danger`, `success`, `outline`, `ghost`, `link` |
| `size` | `Size` | `"md"` | One of `sm`, `md`, `lg`, `icon` |
| `icon` | `string` | — | Iconify class for left icon |
| `iconRight` | `string` | — | Iconify class for right icon |
| `loading` | `boolean` | `false` | Shows spinner and disables button |
| `disabled` | `boolean` | `false` | Disables button interaction |
| `tooltip` | `string` | — | Tooltip text on hover |
| `shortcut` | `string` | — | Keyboard shortcut shown in tooltip |
| `asChild` | `boolean` | `false` | Merge props onto child element |
| `allowed` | `boolean` | — | Permission gating |
| `reason` | `string` | — | Explains why button is disabled |

### ButtonGroup Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `orientation` | `string` | `"horizontal"` | `horizontal` or `vertical` |
