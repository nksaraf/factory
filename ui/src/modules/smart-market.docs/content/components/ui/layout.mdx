# Layout

Structural components for organizing content into sections, cards, and collapsible regions.

```tsx
import { Card, CardContent, CardDescription, CardFooter, CardHeader, CardTitle } from "@rio.js/ui-next/components/card"
```

---

## Card

A bordered container for grouping related content with an optional header and footer.

<ComponentPreview className="block">
  <Card className="w-full max-w-sm">
    <CardHeader>
      <CardTitle>Card Title</CardTitle>
      <CardDescription>Card description with supporting text.</CardDescription>
    </CardHeader>
    <CardContent>
      <p className="text-sm">This is the card body content. It can contain any elements.</p>
    </CardContent>
    <CardFooter className="flex justify-end gap-2">
      <Button variant="outline" size="sm">Cancel</Button>
      <Button size="sm">Save</Button>
    </CardFooter>
  </Card>
</ComponentPreview>

```tsx
<Card>
  <CardHeader>
    <CardTitle>Card Title</CardTitle>
    <CardDescription>Description text.</CardDescription>
  </CardHeader>
  <CardContent>Content here</CardContent>
  <CardFooter>Footer actions</CardFooter>
</Card>
```

### Card structure

| Component | Purpose |
|-----------|---------|
| `Card` | Root container with border and background |
| `CardHeader` | Top section with padding for title and description |
| `CardTitle` | Heading text |
| `CardDescription` | Supporting text below the title |
| `CardContent` | Main body area |
| `CardFooter` | Bottom section, typically for actions |

### Card without header

For simpler use cases, use `CardContent` directly.

```tsx
<Card>
  <CardContent className="pt-6">
    Simple card with just content.
  </CardContent>
</Card>
```

## Separator

A visual divider between sections of content. Renders as an `<hr>` with consistent styling.

```tsx
import { Separator } from "@rio.js/ui-next/components/separator"
```

<ComponentPreview>
  <div className="space-y-4 w-full max-w-sm">
    <div>
      <h4 className="text-sm font-medium">Section One</h4>
      <p className="text-sm text-muted-foreground">First section content.</p>
    </div>
    <Separator />
    <div>
      <h4 className="text-sm font-medium">Section Two</h4>
      <p className="text-sm text-muted-foreground">Second section content.</p>
    </div>
  </div>
</ComponentPreview>

```tsx
<Separator />
```

### Vertical separator

Use `orientation="vertical"` in flex layouts to divide horizontal content.

```tsx
<div className="flex items-center gap-4 h-8">
  <span>Left</span>
  <Separator orientation="vertical" />
  <span>Right</span>
</div>
```

## Accordion

Collapsible sections for organizing content that doesn't need to be visible all at once. Ideal for FAQs, settings groups, and dense information.

```tsx
import { Accordion, AccordionContent, AccordionItem, AccordionTrigger } from "@rio.js/ui-next/components/accordion"
```

<ComponentPreview className="block">
  <Accordion type="single" collapsible className="w-full max-w-sm">
    <AccordionItem value="item-1">
      <AccordionTrigger>What is this?</AccordionTrigger>
      <AccordionContent>
        A collapsible section for organizing related content.
      </AccordionContent>
    </AccordionItem>
    <AccordionItem value="item-2">
      <AccordionTrigger>How do I use it?</AccordionTrigger>
      <AccordionContent>
        Wrap your content in AccordionItem with a trigger and content.
      </AccordionContent>
    </AccordionItem>
    <AccordionItem value="item-3">
      <AccordionTrigger>Can I open multiple?</AccordionTrigger>
      <AccordionContent>
        Set type="multiple" to allow multiple items open at once.
      </AccordionContent>
    </AccordionItem>
  </Accordion>
</ComponentPreview>

```tsx
<Accordion type="single" collapsible>
  <AccordionItem value="item-1">
    <AccordionTrigger>Question</AccordionTrigger>
    <AccordionContent>Answer</AccordionContent>
  </AccordionItem>
</Accordion>
```

### Single vs Multiple

| Type | Behavior |
|------|----------|
| `single` | Only one item open at a time. Add `collapsible` to allow closing all |
| `multiple` | Any number of items can be open simultaneously |

```tsx
{/* Only one open at a time, can close all */}
<Accordion type="single" collapsible>

{/* Multiple open at once */}
<Accordion type="multiple">
```

## Scroll Area

A custom scrollable container with styled scrollbars that match the design system.

```tsx
import { ScrollArea } from "@rio.js/ui-next/components/scroll-area"
```

<ComponentPreview>
  <ScrollArea className="h-48 w-48 rounded-md border p-4">
    {Array.from({ length: 20 }, (_, i) => (
      <div key={i} className="py-1 text-sm">
        Item {i + 1}
      </div>
    ))}
  </ScrollArea>
</ComponentPreview>

```tsx
<ScrollArea className="h-48 w-48 rounded-md border p-4">
  {items.map(item => <div key={item.id}>{item.name}</div>)}
</ScrollArea>
```

### Horizontal scrolling

```tsx
<ScrollArea className="w-full" orientation="horizontal">
  <div className="flex gap-4">
    {items.map(item => <Card key={item.id} className="w-64 shrink-0" />)}
  </div>
</ScrollArea>
```

## API Reference

### Card Props

All card sub-components extend native HTML `div` attributes.

| Component | Key props | Description |
|-----------|-----------|-------------|
| `Card` | `className` | Root container |
| `CardHeader` | `className` | Title and description area |
| `CardTitle` | `className` | Heading text |
| `CardDescription` | `className` | Supporting text |
| `CardContent` | `className` | Main content area |
| `CardFooter` | `className` | Footer actions area |

### Separator Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `orientation` | `"horizontal" \| "vertical"` | `"horizontal"` | Direction of the divider |
| `decorative` | `boolean` | — | If true, hidden from assistive technology |

### Accordion Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `type` | `"single" \| "multiple"` | — | Whether one or many items can be open |
| `collapsible` | `boolean` | `false` | Allow closing all items (single mode only) |
| `value` | `string \| string[]` | — | Controlled open item(s) |
| `defaultValue` | `string \| string[]` | — | Uncontrolled default |
| `onValueChange` | `(value) => void` | — | Called when open items change |

### ScrollArea Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `orientation` | `"vertical" \| "horizontal"` | `"vertical"` | Scroll direction |
| `className` | `string` | — | Must include a fixed height/width for scrolling |
