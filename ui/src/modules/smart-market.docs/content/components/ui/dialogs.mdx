# Dialogs

Modal and overlay components for focused interactions. Built on Radix Dialog primitives with consistent animations and accessibility.

```tsx
import { Dialog, DialogContent, DialogDescription, DialogFooter, DialogHeader, DialogTitle, DialogTrigger } from "@rio.js/ui-next/components/dialog"
```

---

## Dialog

A centered modal overlay for forms, confirmations, and focused workflows. Traps focus and dims the background.

<ComponentPreview>
  <Dialog>
    <DialogTrigger asChild>
      <Button variant="outline">Open Dialog</Button>
    </DialogTrigger>
    <DialogContent>
      <DialogHeader>
        <DialogTitle>Edit Profile</DialogTitle>
        <DialogDescription>Make changes to your profile here.</DialogDescription>
      </DialogHeader>
      <div className="space-y-4 py-4">
        <div className="space-y-2">
          <Label htmlFor="name">Name</Label>
          <Input id="name" defaultValue="Nikhil Saraf" />
        </div>
        <div className="space-y-2">
          <Label htmlFor="email-dialog">Email</Label>
          <Input id="email-dialog" defaultValue="nikhil@saraf.com" />
        </div>
      </div>
      <DialogFooter>
        <Button variant="outline">Cancel</Button>
        <Button>Save changes</Button>
      </DialogFooter>
    </DialogContent>
  </Dialog>
</ComponentPreview>

```tsx
<Dialog>
  <DialogTrigger asChild>
    <Button variant="outline">Open Dialog</Button>
  </DialogTrigger>
  <DialogContent>
    <DialogHeader>
      <DialogTitle>Edit Profile</DialogTitle>
      <DialogDescription>Make changes to your profile here.</DialogDescription>
    </DialogHeader>
    {/* Form content */}
    <DialogFooter>
      <Button variant="outline">Cancel</Button>
      <Button>Save changes</Button>
    </DialogFooter>
  </DialogContent>
</Dialog>
```

### Dialog structure

| Component | Purpose |
|-----------|---------|
| `Dialog` | Root — manages open/close state |
| `DialogTrigger` | Element that opens the dialog |
| `DialogContent` | The modal panel itself |
| `DialogHeader` | Container for title and description |
| `DialogTitle` | Heading text (required for accessibility) |
| `DialogDescription` | Supporting text below the title |
| `DialogFooter` | Action buttons area, right-aligned |

### Controlled dialog

```tsx
const [open, setOpen] = useState(false)

<Dialog open={open} onOpenChange={setOpen}>
  <DialogContent>
    {/* ... */}
    <DialogFooter>
      <Button onClick={() => setOpen(false)}>Done</Button>
    </DialogFooter>
  </DialogContent>
</Dialog>
```

## Alert Dialog

A confirmation dialog that requires an explicit user decision. Unlike Dialog, it cannot be dismissed by clicking the overlay or pressing Escape — the user must choose an action.

```tsx
import { AlertDialog, AlertDialogAction, AlertDialogCancel, AlertDialogContent, AlertDialogDescription, AlertDialogFooter, AlertDialogHeader, AlertDialogTitle, AlertDialogTrigger } from "@rio.js/ui-next/components/alert-dialog"
```

<ComponentPreview>
  <AlertDialog>
    <AlertDialogTrigger asChild>
      <Button variant="destructive">Delete Item</Button>
    </AlertDialogTrigger>
    <AlertDialogContent>
      <AlertDialogHeader>
        <AlertDialogTitle>Are you sure?</AlertDialogTitle>
        <AlertDialogDescription>
          This action cannot be undone. This will permanently delete the item from the system.
        </AlertDialogDescription>
      </AlertDialogHeader>
      <AlertDialogFooter>
        <AlertDialogCancel>Cancel</AlertDialogCancel>
        <AlertDialogAction>Delete</AlertDialogAction>
      </AlertDialogFooter>
    </AlertDialogContent>
  </AlertDialog>
</ComponentPreview>

```tsx
<AlertDialog>
  <AlertDialogTrigger asChild>
    <Button variant="destructive">Delete</Button>
  </AlertDialogTrigger>
  <AlertDialogContent>
    <AlertDialogHeader>
      <AlertDialogTitle>Are you sure?</AlertDialogTitle>
      <AlertDialogDescription>This cannot be undone.</AlertDialogDescription>
    </AlertDialogHeader>
    <AlertDialogFooter>
      <AlertDialogCancel>Cancel</AlertDialogCancel>
      <AlertDialogAction>Confirm</AlertDialogAction>
    </AlertDialogFooter>
  </AlertDialogContent>
</AlertDialog>
```

### When to use Dialog vs Alert Dialog

- **Dialog** — Forms, settings, content previews. Can be dismissed freely.
- **Alert Dialog** — Destructive or irreversible actions. Forces an explicit choice.

## Sheet

A panel that slides in from the edge of the screen. Use for secondary navigation, detail views, or forms that don't warrant a full modal.

```tsx
import { Sheet, SheetContent, SheetDescription, SheetHeader, SheetTitle, SheetTrigger } from "@rio.js/ui-next/components/sheet"
```

<ComponentPreview>
  <div className="flex gap-2">
    <Sheet>
      <SheetTrigger asChild>
        <Button variant="outline">Open Right</Button>
      </SheetTrigger>
      <SheetContent>
        <SheetHeader>
          <SheetTitle>Side Panel</SheetTitle>
          <SheetDescription>A slide-out panel from the edge of the screen.</SheetDescription>
        </SheetHeader>
        <div className="py-4">
          <p className="text-sm text-muted-foreground">Panel content goes here.</p>
        </div>
      </SheetContent>
    </Sheet>
    <Sheet>
      <SheetTrigger asChild>
        <Button variant="outline">Open Left</Button>
      </SheetTrigger>
      <SheetContent side="left">
        <SheetHeader>
          <SheetTitle>Left Panel</SheetTitle>
          <SheetDescription>Opens from the left side.</SheetDescription>
        </SheetHeader>
      </SheetContent>
    </Sheet>
  </div>
</ComponentPreview>

```tsx
<Sheet>
  <SheetTrigger asChild>
    <Button>Open Sheet</Button>
  </SheetTrigger>
  <SheetContent side="right">
    <SheetHeader>
      <SheetTitle>Title</SheetTitle>
      <SheetDescription>Supporting text.</SheetDescription>
    </SheetHeader>
    {/* Content */}
  </SheetContent>
</Sheet>
```

### Side options

| Side | Description |
|------|-------------|
| `right` | Default. Slides in from the right edge |
| `left` | Slides in from the left edge |
| `top` | Slides down from the top |
| `bottom` | Slides up from the bottom |

## API Reference

### Dialog Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `open` | `boolean` | — | Controlled open state |
| `defaultOpen` | `boolean` | `false` | Uncontrolled default |
| `onOpenChange` | `(open: boolean) => void` | — | Called when open state changes |
| `modal` | `boolean` | `true` | Whether to render as a modal |
| `variant` | `"default" \| "loud"` | `"default"` | `loud` adds a prominent ring highlight around the dialog |

### AlertDialog Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `open` | `boolean` | — | Controlled open state |
| `defaultOpen` | `boolean` | `false` | Uncontrolled default |
| `onOpenChange` | `(open: boolean) => void` | — | Called when open state changes |

### SheetContent Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `side` | `"top" \| "right" \| "bottom" \| "left"` | `"right"` | Edge the sheet slides from |
| `className` | `string` | — | Additional CSS classes |
